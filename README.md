# Obsidian Web Clipper

A Chrome extension (Manifest V3) that lets you capture web pages and selections directly into your [Obsidian](https://obsidian.md) vault, with a built-in ProseMirror markdown editor, AI-powered tag suggestions, and full vault search — all from a persistent browser side panel.

![Editor tab showing a captured note with tags and folder](https://user-images.githubusercontent.com/placeholder/screenshot.png)

---

## Features

### ✂️ Web Capture
- **Capture full page** — extracts the main article content and converts it to clean Markdown
- **Capture selection** — right-click any selected text and send it to the editor
- Both modes auto-populate frontmatter: source URL, page title, and capture timestamp
- Accessible from the side panel toolbar or the right-click context menu

### 📝 ProseMirror Editor
- Rich markdown editor powered by [ProseMirror](https://prosemirror.net)
- Frontmatter (title, tags, folder) shown in the header — not cluttering the editor body
- **Autosave** — debounced 1.5 s after each edit, writes directly to the vault file
- **Discard** button to abandon a captured draft

### 🗂️ Vault Integration
- Connects to the [Obsidian Local REST API](https://github.com/coddingtonbear/obsidian-local-rest-api) community plugin
- Supports both **plain HTTP** (`http://127.0.0.1:27123`, recommended) and **HTTPS** (`https://127.0.0.1:27124`)
- **Folder dropdown** — live-filtered combobox with 250 ms debounce, lazily loads all vault folders
- **Save to Vault** — creates or updates the note file via `PUT /vault/<path>`
- **Open in Obsidian** — launches the note in the desktop app via the `obsidian://` URI scheme
- **Sync from Vault** — pulls the latest version; on conflict saves your editor copy as `<name>.conflict.<ts>.md` and shows a resolution banner

### 🔍 Search
- Full-text vault search via `POST /search/simple/?query=`
- Results listed with filename and folder path, clickable to open in the editor

### 🏷️ AI Tag Suggestions
- One-click **Suggest** button sends note content to your configured LLM
- Returns 3–7 lowercase kebab-case tags as removable chips
- Tags stored in YAML frontmatter, synced back to the vault on save

### ⚙️ Settings
- **Vault**: Local REST API URL and Bearer token
- **LLM**: Provider (OpenAI, Google Gemini, DeepSeek, or any OpenAI-compatible endpoint), API key, model name, and optional custom endpoint URL
- Credentials stored in `chrome.storage.local` — never synced, never leaves the service worker

## Installation

### From source

```bash
git clone https://github.com/your-username/obsidian-web-clipper.git
cd obsidian-web-clipper
npm install
npm run build
```

Then load the extension in Chrome:

1. Go to `chrome://extensions`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked**
4. Select the repository root folder

### Connect to Obsidian

1. In Obsidian, open **Settings → Community plugins → Browse**, search for **Local REST API**, install and enable it
2. In the plugin settings, copy your **API key**
3. Open the Obsidian Clipper side panel in Chrome (click the extension icon or toolbar button)
4. Go to the **Settings** tab
5. Set the URL:
   - Plain HTTP (recommended, no cert required): `http://127.0.0.1:27123`
   - HTTPS (self-signed cert — visit the URL in a tab first and accept the certificate): `https://127.0.0.1:27124`
6. Paste your API key and click **Test Connection**
7. Optionally configure your LLM provider for AI tag suggestions

## Architecture

```
Browser tab
    │  content.js (injected)
    │  extracts HTML → Markdown
    ▼
background.js (MV3 Service Worker)
    ├── routes messages (captureCurrentTab, vault:*, llm:*)
    ├── vaultFetch() — direct fetch() to 127.0.0.1 (host_permissions covers it)
    └── LLMClient — calls provider APIs with key from chrome.storage.local
    ▼
sidebar/index.html (Side Panel)
    ├── Tab: Editor  — ProseMirror, frontmatter bar (title / folder / tags), sync, autosave
    ├── Tab: Recent  — vault file list, click to open in editor
    ├── Tab: Search  — full-text vault search
    └── Tab: Settings — vault URL/key, LLM provider/key/model
```

**Why direct `fetch()` from the service worker?**  
Earlier iterations injected a script into a tab via `chrome.scripting.executeScript` to work around a service-worker TLS limitation. That approach required broad `host_permissions` and caused _"Cannot access contents of the page"_ errors. The current approach uses `fetch()` directly from the service worker, which works cleanly because `127.0.0.1` is in `host_permissions`. For HTTPS with a self-signed cert, the user accepts it once in a browser tab; after that Chrome's network stack trusts it profile-wide.

## Development

```bash
# Install dependencies
npm install

# Build sidebar bundle (esbuild, ~20ms)
npm run build

# Watch mode
npm run build:watch

# Run Playwright tests (headed, Chromium with extension loaded)
npm test

# Open Playwright UI
npm run test:ui
```

### Running tests

Tests use a real Chromium instance with the extension loaded. `chrome.*` APIs are stubbed via `addInitScript` so vault and LLM calls never leave the test environment.

```
72 tests  ·  ~11 s
```

Key test areas:
- Tab navigation and UI state
- Frontmatter parsing (inline array tags, YAML block list tags, title extraction)
- Folder combobox (focus, filter, keyboard nav, debounce)
- Sync conflict detection and resolution
- Search request shape (no `Content-Type` header, `?query=` param)
- Save / autosave / discard flow
- LLM tag suggestion message shape
- Settings form and connection test

## Note Format

Notes are saved as standard Obsidian Markdown with YAML frontmatter:

```markdown
---
title: "My Note Title"
tags: ["web", "research", "ai"]
updated: "2024-11-01T12:00:00.000Z"
---

Note body here…
```

When loading an existing note the frontmatter is parsed and displayed in the UI header (title input, tag chips, folder field). It is **not** shown in the editor body. On save, the frontmatter is rebuilt from the current UI state and prepended to the editor content.

Obsidian's block-list tag format is also supported on load:

```markdown
tags:
  - web
  - research
```

## LLM Providers

| Provider | Default model | Notes |
|---|---|---|
| OpenAI | `gpt-4o-mini` | Standard Chat Completions API |
| Google Gemini | `gemini-1.5-flash` | Uses `v1beta/models/:model:generateContent` |
| DeepSeek | `deepseek-chat` | OpenAI-compatible endpoint |
| OpenAI-compatible | _(custom)_ | Paste any OpenAI-compatible endpoint URL |

API keys are stored in `chrome.storage.local` and only used from the service worker — they are never accessible from page-context scripts.

## Permissions

| Permission | Why |
|---|---|
| `sidePanel` | Persistent side panel UI |
| `storage` | Save settings and credentials locally |
| `activeTab` | Capture the current page via content script |
| `contextMenus` | Right-click "Capture page / selection" menu items |
| `host_permissions: 127.0.0.1, localhost` | Direct `fetch()` to the Obsidian Local REST API |
